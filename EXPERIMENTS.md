## Environment (proven, on PCSS Eagle)

- **Cluster/SLURM:** account `pl0896-03`, QOS `normal,tesla`. GPU jobs ‚Üí `--partition=tesla --gres=gpu:h100:1` (GRES type, **not** `--constraint`). CPU-only ‚Üí `--partition=standard`.
- **venv (`.venv/`, git-ignored):** built on **Python 3.9** (faiss-gpu wheels don't cover 3.13); `venv` ensurepip is broken here, so:
  ```bash
  python3 -m venv --without-pip .venv
  ```

- **TORCH_HOME=`data/torch_home`** caches the SWSL hub repo + weights ‚Üí compute nodes need no internet.
- **torch 2.8 quirk:** loading our checkpoints needs `torch.load(..., weights_only=False)` (they store a numpy scalar) ‚Äî handled in `extract_descriptors.py` and the `--init_weights` path.

### EXP-4 ‚Äî step 4, train/fine-tune WITH synthetic data üü°
Question: does adding the synthetic gallery data to training improve recognition of **real painting photos** (step-2 set) without hurting the rest? Eval DB stays the original studio set ‚Üí directly comparable.

| Run | Recipe | Data | Job(s) | Status |
|---|---|---|---|---|
| Baseline | from-SWSL, 10 ep | studio only | (step 1) | ‚úÖ GAP 35.97 |
| Synth-only FT | fine-tune ep10, 5 ep @1e-7 | synthetic only | train 7330026 / eval 7330036 | ‚úÖ |
| Combined FT | fine-tune ep10, 5 ep @1e-7 | studio + synthetic | 7330025 | üü° running |
| **From-scratch +synth** | from-SWSL, 10 ep | studio + synthetic | 7330059 | üü° running ‚Äî **clean A/B vs step 1** |

Fine-tunes load epoch-10 weights via **`--init_weights`** (added to `train_contrastive.py`) with a fresh optimizer @ **LR 1e-7** ‚Äî a literal `--resume` runs at the decayed **1e-8 ‚âà frozen** (no-op). `finetune.slurm <combined|synth>`; from-scratch is `train_synth.slurm`. Eval: `extract_eval_ft.slurm <variant>`.

**Synth-only FT result** (original studio DB + real queries; best K=15, œÑ=50):

| | Full GAP | GAP‚Åª | ACC | Paint GAP‚Åª (strict) | Paint ACC (strict) |
|---|--:|--:|--:|--:|--:|
| Baseline (step 1) | 35.97 | 52.14 | 54.64 | 67.81 | 69.36 |
| **Synth-only FT** | **38.61** | **55.59** | **57.83** | **72.79** | **73.99** |
| Œî | +2.64 | +3.45 | +3.19 | +4.98 | +4.63 |

(broad paintings: GAP‚Åª 65.92‚Üí70.41, ACC 67.42‚Üí71.49.)

**Finding:** synth-only fine-tuning improved *everything*, including **non-paintings** (+2.6 full GAP) ‚Äî surprising; the prediction was forgetting. Because non-paintings also rose, it's not pure forgetting ‚Äî the diverse renders taught **transferable lighting/viewpoint/glass invariance** that helps real photos broadly. **‚öÝÔ∏è Confound:** we re-warmed LR (1e-8‚Üí1e-7) + trained 5 extra epochs, so part of the gain may be extra training, not synthetic. **Resolution:** the **from-scratch +synth** run (identical 10-epoch recipe to step-1, only synthetic added) is the confound-free A/B ‚Äî that's the headline number to report; it also removed the need for a separate control run.

### EXP-5 ‚Äî step 5, new method üü°
**Additional experiment ‚Äî cross-domain pair mining.** New `--pairs_type cross_domain_pos+new_neg`
(`code/utils/datasets.py`): each anchor's positive is the *closest same-class sample in the OTHER
domain* (studio‚Üîsynthetic) via `mine_positive` on the cross-domain subset; classes with no
cross-domain partner fall back to standard `new_pos`, singletons self-pair. Directly optimizes the
studio‚Üígallery-photo bridge. Fires for the **4,952** painting classes that have both a studio and a
synthetic image (verified on `data/gt_aug`). Mining the *closest* render also sidesteps the broken
grazing-view renders (they sit far from the studio image). Recipe = `train_synth.slurm` (from-SWSL,
10 ep, combined manifest) with only the pairs_type changed ‚Üí **clean A/B vs scratch+synth** (job
7330059). Job `train_crossdomain.slurm` ‚Üí **7332307 (queued**, waiting on a GPU). Mining logic
smoke-tested (tiny mixed manifest, asserts cross/fallback/singleton/negatives). Eval TBD (epoch-10
ckpt ‚Üí `data/models/r18SWSL_crossdomain/`; needs the same eval tweak as scratch+synth, not the
`extract_eval_ft.slurm` epoch-5 globber).

## Gotchas & decisions
- **Eval must tune the full K grid** (`scripts/eval_fullgrid.py`), not the README's `--autotune` (K=1, ‚Üí GAP ‚âà 23).
- `--net r18_sw-sup` for training, `r18_contr_loss_gem_fc_swsl` (+`--netpath`) for extraction.
- faiss on **CPU** (H100 sm_90 gap); identical exact-IP results.
- **Checkpoint load needs `weights_only=False`** (torch 2.8).
- **Fine-tune LR:** load weights via `--init_weights` @ 1e-7; literal `--resume` = decayed 1e-8 = no-op.
- **Eval DB = original 397k studio images** for every run (so all models are comparable; synthetic is added to *training* only, never the DB).
- **Synthetic camera-rig bug:** `right upper` grazing view (R@1 1.4%) ‚Äî fix + regenerate before per-angle synthetic claims.
- GPU = `--gres=gpu:h100:1` on `tesla`. Authors' artifacts under `data/authors/` (separate from our `data/models/`).
